## Card Grid Layout

- **Landscape**: 3 columns
- **Portrait**: 2 columns
- Cards use the **Starter Card** shared component (see 07-shared-components.md)
- Grid scrolls vertically if more starters than fit on screen
- Cards are equal height within each row

## Card Content

Each card displays (via the Starter Card component):
1. **Health dot** вҖ” рҹҹў Thriving (< 12h), рҹҹЎ Needs Feeding (1-7 days), рҹ”ҙ Needs Revival (> 7 days)
2. **Starter name** вҖ” Fraunces serif, prominent
3. **Time since last feed** вҖ” "Fed 6 hours ago" or "Never fed" for new starters
4. **Last feeding flour summary** вҖ” "AP + Rye В· 150g each"
5. **Last peak time** вҖ” "Peaked at 4.5h" (if recorded)
6. **Thumbnail photo** вҖ” from most recent feeding (placeholder if none)

## "+ New Starter" Button

- Top-right of the screen
- Navigates to **Create a Starter** form (13-create-starter.md)
- After creation, the new starter card appears in the grid

## Interactions

- **Tap a card** вҶ’ push navigation to **Starter Details** (11-starter-details.md)
- **Long-press a card** вҶ’ context menu:
  - "Delete Starter" (with confirmation alert: "Delete [name]? This will also delete all feeding history.")
- **Pull to refresh** вҖ” updates health indicators and time-since-feed
- **"+ New Starter"** вҶ’ Create a Starter form

## Single Starter Optimization

When there's only one starter, the card takes full width (centered, max ~400px)
and the layout feels less like a grid and more like a focused detail preview.
